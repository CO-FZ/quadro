## Decisões pequenas tomadas

- Cor violet para em_revisao: neutro entre primary (teal/blue) e green (finalizada) — semanticamente "em avaliação" antes de "concluída".
- Grid Kanban: `sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-5` em vez de `lg:grid-cols-5` direto — 5 colunas em 1024px ficaria muito estreito.
- View SQL adicionou `in_review_tasks` além de `avatar_url` e `full_name` — view estava omitindo full_name (presente na interface mas ausente no SELECT anterior).
- `ALTER TYPE ... ADD VALUE` rodado sem BEGIN/COMMIT conforme padrão da migration arquivada anterior (Postgres não suporta ADD VALUE em transação).
- Avatar fallback: `s.avatar_url` null/undefined → círculo com inicial. Sem `<Image>` do Next.js para evitar configuração extra de `remotePatterns` (mesmo padrão de ProfileView.tsx).
