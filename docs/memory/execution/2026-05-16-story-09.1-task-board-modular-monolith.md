# Sessão Story 09.1 — Task Board Modular Monolith — 2026-05-16

## Plano original

Extrair `task-board/domain`, `application`, `infrastructure` e tornar `lib/actions/tasks.ts` facade fina. Eliminar duas fontes de verdade para tipos de tarefa. Criar `TaskRepository` port para tornar use cases testáveis sem Supabase real.

## Desvios do plano

- `lib/actions/tasks.ts` já era facade fina — nenhuma alteração necessária (passo 11 do plano eliminado)
- `domain/task.ts` não foi deletado: convertido para re-exporter para manter imports de testes funcionando

## Decisões pequenas tomadas

- Regra de negócio backlog→alocada (quando assignees são adicionados) estava em `SupabaseTaskRepository.updateTaskAssignees` — violação do ADR 0006 (infra não pode conter regras de domínio). Movida para `TaskUseCases.updateTaskAssignees`.
- `lib/supabase/types.ts` precisou de `import type { AppRole }` explícito além do re-export: TypeScript não expõe re-exports como bindings locais.
- `pnpm lint` falhou por interferência do hook rtk no JSON output. Workaround: `node_modules/.bin/eslint` direto — zero erros.
